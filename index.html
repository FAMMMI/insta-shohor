1. How does Javascript works?

Answer: Javascript is a single-threaded and single concurrent programming language means it can handle one task at a time. Javascript is designated to ensure the non-blocking execution. It doesn't prevent the execution of another piece of code rather it waits in a loop for a network request and accept incoming request after the first one is handled. It uses event loop to execute an event where the synchronous events are stored in a stack and asynchronous are stored in a queue. When the execution starts, first the events are executed from stacks and then from the queue when the stack becomes empty. As it is a client-side scripting language, it runs at the client-side web browser which supports javascript. For example Google Chrome uses V8 engine to run javascript.

2.Differenece between local storage and session storage .

Answer: The storage API consists of two mecahnism :
Local Storage and
Session Storage.
Both of them maintain a separate storage area for each available origin for the time duration of the page session. The differences between Local and Session Storage are given below:
** Local storage is not session based, it should be deleted from javascript manally where Session storage works per window or tab means data is stored until the browser or tab is closed.
** The client can only read Local Storage , not session storage.
** The Storage capacity of local storage is 5MB or 10 MB but capacity of session storage is 5MB.
